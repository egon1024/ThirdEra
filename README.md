# ThirdEra
A D&amp;D 3.5 game system for Foundry VTT

**[Documentation](https://egon1024.github.io/ThirdEra/latest/)** (versioned; select a release in the nav if needed).

Any images in this system that aren’t from Foundry were generated by AI—none of the artwork was created by me or anyone else.

## Compendiums

ThirdEra includes compendium packs with SRD content to make character creation
immediate and easy. All compendiums are available in the Foundry VTT Compendium
sidebar:

- **Races** - All SRD races (Dwarf, Elf, Gnome, Half-Elf, Half-Orc, Halfling, Human)
- **Classes** - All SRD base classes (Barbarian, Bard, Cleric, Druid, Fighter, Monk, Paladin, Ranger, Rogue, Sorcerer, Wizard)
- **Skills** - All SRD skills (35+ skills with proper ability scores and flags)
- **Feats** - Core SRD feats with prerequisites
- **Weapons** - Common SRD weapons (simple, martial, exotic)
- **Armor** - SRD armor and shields with full stats
- **Equipment** - Adventuring gear and containers
- **Spells** - Core SRD spells across multiple levels

## Automated tests (contributors)

From the repository root, with **Node.js 20+**: run `npm ci` once (or `npm install`), then `npm test` (fast) or **`npm run test:coverage`** (tests + report in `coverage/`; open `coverage/index.html`). **Validate** on GitHub runs **`test:coverage`** and attaches a **`coverage-report`** artifact. **What is / is not covered:** [`test/README.md`](test/README.md). See **Development** → **Automated unit tests** in the [documentation](https://egon1024.github.io/ThirdEra/latest/).

## Documentation

The user-facing documentation site is built with [MkDocs](https://www.mkdocs.org/), [Material for MkDocs](https://squidfunk.github.io/mkdocs-material/), and [mike](https://github.com/jimporter/mike) for versioned deploys. Source lives in `docs-site/`; config is `mkdocs.yml`. To work on it locally:

1. **Install:** From the repo root, `pip install -r requirements-docs.txt`.
2. **Preview:** Run `mkdocs serve` (default: http://127.0.0.1:8000). Edit files in `docs-site/`; the browser live-reloads.
3. **Build only:** Run `mkdocs build` to output the site to `site/` (e.g. for link checks or local inspection).
4. **Versioned deploy:** The release workflow runs `mike deploy` with the release version on merge, so docs and system versions stay in sync with the GitHub release. Run `mike set-default --push latest` once so the root URL redirects to latest.
